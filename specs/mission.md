# BookWorm Engineer - Mission

## What We Are Building

BookWorm Engineer is an AI-powered coding assistant that combines the research capabilities of NotebookLM with the interactive CLI experience of opencode. It helps developers analyze project sources, plan solutions, implement code, and maintain project continuity through a structured workflow.

## Why We Are Building This

Traditional AI coding assistants often lack:

1. **Research-grounded responses** - They don't cite sources or verify facts against project documentation
2. **Structured project workflows** - No clear separation between planning, building, and research phases
3. **Persistent project state** - No way to track progress across multiple interactions
4. **Source-based knowledge** - Limited to internet knowledge rather than project-specific context

BookWorm Engineer solves these problems by:

- **Indexing project sources** (PDFs, text files, markdown) for context-aware responses
- **Maintaining project continuity** through PROGRESS.md and AGENTS.md files
- **Supporting three distinct modes** (plan, build, research) for different types of tasks
- **Providing verifiable answers** with citations to source documents
- **Automating project management** through git commits and progress tracking

## Our Vision

We envision BookWorm Engineer as the ultimate coding companion that:

- Acts as both a research assistant and implementation partner
- Maintains context across multiple sessions through project indexing
- Follows established project patterns and guardrails
- Helps teams maintain consistent coding practices
- Reduces cognitive load by handling repetitive project management tasks

The goal is to create an AI assistant that doesn't just write code, but understands the project's context, history, and requirements to make intelligent, context-aware decisions.
